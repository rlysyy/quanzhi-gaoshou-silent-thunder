## 🔄 版本管理方案

### 提交规范
```
v{主版本}.{次版本}: {更新内容简述}
```

### 版本记录
```bash
# 查看提交历史
git log --oneline

# 查看当前版本
git log -1

# 比较版本差异
git diff HEAD~1
```

### 工作流
1. **修改文档** → 编辑 `全职高手续写_无声惊雷_续写.md`

3. **飞书发送** → 将最新版本发送给陛下审阅
4. **根据反馈修改** → 再次提交新版本
